EOS Certification Website

The Concept

EOSCert Belgium is a comprehensive certification management platform designed to handle the certification processes for EOS standards. The primary interface is for companies and the public. It serves as an informational hub and a functional tool for requesting quotes, submitting complaints, and verifying certificates.

Key Features

Public Portal:

  • Dynamic Quote Request Engine: A complex, multi-step form that captures detailed organizational data (standards, audit scope, employee counts). Upon submission, the backend dynamically generates a formatted Microsoft Word (`.docx`) document containing all the data and emails it to the administration for review.
  • Certificate Verification: A public tool that allows anyone to verify the validity of a certificate. Users can scan a QR code. The system validates this against the database and returns the certificate status, ensuring transparency and trust.
  • Contact & Complaint Management: Dedicated forms for general inquiries and formal complaints. These forms are protected by reCAPTCHA and trigger immediate email notifications to the support team.
  • Internationalization: The site is built with multi-language support in mind, utilizing `Transloco` to serve content in different languages.

Backend Infrastructure

  • Server-Side Rendering (SSR) with CSP: The application is served via a Cloud Function that injects a strict, nonce-based Content Security Policy (CSP) into the `index.html` at runtime. This provides a high level of security against Cross-Site Scripting (XSS) attacks.
  • Automated Communication: Extensive use of transactional emails (via `Nodemailer` and Gmail) to keep all stakeholders informed about application status, quotes, and inquiries.

Technology

Frontend

  • Framework: Angular 19
  • Hosting: Firebase Hosting
  • State & Async management: RxJS
  • UI/UX Libraries
    • Swiper: For responsive carousels and sliders.
    • Transloco: For handling translations and internationalization.
    • qrcode: For generating and handling QR codes client-side.
  • Security: Google reCAPTCHA v3

Backend

  • Runtime: Node.js 20

    • docx: For programmatic generation of Word documents.
    • nodemailer: For sending SMTP emails via Gmail.
    • axios: For making external HTTP requests (e.g., reCAPTCHA verification).

EOS Certification Database

The Concept

The EOSCert Belgium platform relies on Cloud Firestore, a scalable NoSQL document database, to manage critical data such as certification records, user applications, and dynamic content. The database is designed to support real-time updates and strict security rules.

Data Management Strategy

The database is architected to handle distinct domains of data while ensuring isolation and performance:

  • Certification Records: Stores comprehensive details about certified companies and their validity status. This data serves as the single source of truth for the public verification tool.
  • Admin Onboarding: Manages the lifecycle of administrator access requests, tracking the status from initial application to final approval or denial.

Security & Access Control

Security is enforced via Firestore Security Rules, implementing a Role-Based Access Control (RBAC) model:

  • Role-Based Access: Only users with specific custom claims (`ADMIN`, `OWNER`) can read or write sensitive documents.
  • Least Privilege: Public access is completely restricted at the database level. Public data retrieval (e.g., certificate verification) is proxied exclusively through secure Cloud Functions (`getCertificateData`), ensuring no direct client-side access to the DB.

Privacy & Compliance (GDPR)

To adhere to privacy regulations like GDPR, the architecture implements several safeguards:

  • Data Minimization: Only essential business data is stored. Personal data is limited to what is strictly necessary for certification and contact purposes.
  • Access Control: Strict RBAC ensures that only authorized personnel can access sensitive records.
  • Indirect Access: Public users cannot query the database directly. All public-facing data is filtered and sanitized by backend Cloud Functions before being returned to the client, preventing accidental exposure of private fields.
  • Encryption: Data is encrypted at rest by Google Cloud Platform standards and in transit via HTTPS.

Integration & Automation

The database acts as an event bus for serverless logic:

  • Triggers: Changes in the 'user-applications' collection (e.g., status change to 'approved') automatically trigger Cloud Functions to create user accounts and send email notifications.
  • Cloud Functions: The 'getCertificateData' function queries 'CertificationApplications' to validate certificates based on complex ID parsing logic.
